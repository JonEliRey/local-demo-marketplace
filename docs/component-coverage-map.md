# Component Coverage Map

Central map for humans and agents implementing the registry-driven marketplace loop.

## Process loop

`registry.yaml` -> `scripts/build.py` -> generated runtime outputs + `build/provenance/*.json` -> verification gates -> catalog/docs updates.

## Major components

| Component | Role | Implementation path | Docs/backlinks | Verification | Gaps/open items |
|---|---|---|---|---|---|
| Registry | Source of truth for sources, assets, strategies, outputs, exposure, plugin metadata. | `registry.yaml` (`sources`, `assets`) | `docs/registry-reference.md`, `catalog/index.md` | `python3 -m unittest tests.test_marketplace_process -v`, `python3 scripts/cli_uat.py` | Keep additions schema-consistent; no generated schema yet. |
| Builder | Deterministically materializes Copilot assets, Claude plugins, companion tools, marketplace manifest, provenance. | `scripts/build.py` | `docs/verification-guide.md` | `python3 scripts/build.py`, `git diff --exit-code -- .claude-plugin .github/agents .github/instructions plugins companion-tools/marketplace-audit` | Optional refactor as strategies grow. |
| Runtime outputs | Committed consumable artifacts for VS Code/Copilot, Claude Code plugin marketplace, companion CLI. | `.github/agents/`, `.github/instructions/`, `plugins/`, `companion-tools/marketplace-audit/` | `catalog/index.md` | `python3 companion-tools/marketplace-audit/marketplace_audit.py --root .` | VS Code/Copilot client-side UI UAT still required on a workstation. |
| Provenance | Audit trail per asset: sources, ingredients, patches, overlays, recipe inputs, model routing, output files. | `build/provenance/*.json` generated by `scripts/build.py` | `docs/registry-reference.md#provenance-contract` | `python3 companion-tools/marketplace-audit/marketplace_audit.py --root .` | Timestamped generated output; keep provenance regenerated before audits. |
| UAT coverage | Executable scenario coverage for all registered strategies/assets. | `scripts/cli_uat.py` (`SCENARIO_COVERAGE`) | `docs/verification-guide.md#command-guide` | `python3 scripts/cli_uat.py` | Claude CLI validation optional unless `--require-claude`. |
| Check gate | Local/CI deterministic gate combining unit tests, compile, build, audit, UAT, optional Claude validation, diff checks. | `scripts/check_marketplace.py`; `.github/workflows/marketplace-ci.yml` | `docs/verification-guide.md` | `python3 scripts/check_marketplace.py` | Hook/MCP asset packaging not implemented. |
| Executable spec | Tests encode required fields, strategy coverage, generated outputs, plugin manifest alignment, fail-closed checks. | `tests/test_marketplace_process.py` | `docs/verification-guide.md#what-the-tests-prove` | `python3 -m unittest tests.test_marketplace_process -v` | Update tests with every new strategy/asset type. |
| Product/spec packet | Captures requirements, design, tasks, and remaining product decisions. | `specs/internal-agent-marketplace/` | `README.md#implementation-map` | Review docs; no single executable command. | Phase 6 product decisions still open. |
